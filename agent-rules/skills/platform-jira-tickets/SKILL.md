# Platform Jira Tickets

## Overview

Platform team tickets live in the PLAT project. They default to issue type "Story" (or "Bug" when specified).

## When to Use

- Creating new Jira tickets for Platform team work
- Updating existing Platform tickets

## Ticket Structure

- **Project:** PLAT
- **Issue type:** Story (default), Bug (if specified)
- **Parent:** Provided per-ticket if needed
- **Assignee:** Nathan Hadley (`712020:0475fc9d-6d24-4ab1-bb30-a36b82b36b8d`)
- **Team:** Platform
- **Priority:** Medium (default)

## Description Format

Two sections: a prose description and acceptance criteria. Best effort based on the info provided — keep it concise.

```
*Description*
A brief prose summary of the work. What's being done and why.

*Acceptance Criteria*
What done looks like. Bullet points are fine here.
```
